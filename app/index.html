LED Project Enabler

Chromatron is an open source pixel controller designed to make LED pixel projects easy and fun.

The goal of this project is to minimize the technical challenges to producing high quality LED projects and artwork. Chromatron does a lot of work for you so you can spend more time perfecting your project, and less time chasing bugs. Everything is designed to be easy to setup and use, but still be powerful enough to create sophisticated projects.

Ultimately, Chromatron seeks to inspire and enable you to create something that you weren't sure you could do otherwise.

Why use Chromatron?

Chromatron is packed with features that make LED projects awesome:

  • Supports a variety of LED pixel types without additional circuitry or firmware modifications.

    Currently, Chromatron supports the APA102, WS2811, WS2812, WS2801, and analog PWM pixel strips. All pixel mode settings are adjustable in real time, without recompiling firmware or rebooting.

  • Control, program, and configure everything over Wi-Fi.

    Ok, initial setup of the Wi-Fi itself has to be done over USB, but once you do that, it's Wi-Fi the rest of the way!

  • Drive up to 300 pixels per controller.

    You can do quite a lot with 300 pixels. But if you need more, you aren't limited to just one Chromatron!

  • Built in scripting engine.

    Chromatron's FX Script is a simple language designed specifically for pixel graphics. It runs on a customed designed virtual machine directly on Chromatron hardware. Compiled FX scripts are quite small, which enables near instant code reloading. You can even run the compiler in live mode, which reloads your script to target devices every time you hit save on your code editor. The virtual machine and language are pixel-aware, which helps you write less code and helps the processor run as quickly as possible.

  • Built in faders

    Chromatron has built in faders for each pixel, along with global master and submaster dimmers. Dimming curves can be adjusted on the fly. Hue, saturation, and brightness can each be faded individually with different timings. All fader calculations are 16 bits to ensure high quality, especially when using dimming curves. Graphics data is reduced to 8 bit RGB only at the very last instant, when the graphics are transferred to the pixels themselves.

  • Python library

    Chromatron has a Python library to make it easy to integrate Chromatron into other projects. It gives you access to everything Chromatron can do: graphics controls, scripting engine and compiler, firmware updates, device discovery, and more.

  • Real time pixel streaming

    Already have a source of graphics content? FX script is an entirely optional feature. You can stream raw HSV or RGB data over the network using the Python library. Art-Net support is planned to be completed before hardware ships.

  • Command line interface

    Get up and running quickly. The CLI gives you quick access to almost everything the Python library can do.

  • Easy hardware connections

    LED and power connections are made with a screw terminal. Soldering (and unsoldering) risks damaging your electronics. Having a screw terminal makes this simple and worry-free.

  • Wide input voltage range

    Chromatron works with 5 and 12 volt pixel strips.

  • Small form factor

    Just 1.25 by 2 inches. Plus a tiny bit of overhang for the USB micro connector.

  • Open source

    All software and hardware will be released under OSI compatible licenses once the project hits the funding goal.

Goal

This Kickstarter is to raise money to build the first production run of Chromatron hardware. The majority of the timeline will be used for coordinating hardware production, building hardware test systems, creating documentation and tutorial content, and of course bug fixes for any issues that crop up.

The project itself has been under heavy development since August 2016, and most major technical aspects are very usable. The hardware itself has undergone five revisions, and over a dozen hand built examples are running 24/7.

API Examples

Discover, configure, and control devices over Wi-Fi with our Python and Command Line Interface:

Python

              
from chromatron import *
# Discover devices in living room
group = DeviceGroup('living_room')
# Load rainbow effect script
group.load_vm('rainbow.fx')
# Set master dimmer to 50%
group.master_dimmer = 0.5

Command Line

              
# Install Python library
pip install chromatron
# Discover devices in living room
chromatron --query living_room discover
# Load rainbow effect script to previous group
chromatron vm load rainbow.fx
# Set master dimmer to 50%
chromatron keys set gfx_master_dimmer 0.5

Hardware

Technical Specs

I/O Processor

  • Atmel AVR XMega128A4U
  • 32 MHz
  • 8K SRAM
  • 128K Flash

This processor runs the real time OS, handles raw pixel data output, USB, auxiliary IO, and the user file system.

Wi-Fi/Graphics processor

  • Espressif ESP8266
  • 160 MHz
  • 64K RAM

Handles Wi-Fi communications, scripting engine VM, pixel faders, and HSV to RGB colorspace conversion. Connected to the real time processor via 4 MHz UART bridge.

I/O

  • 4x PWM outputs, 3 amps each
  • 1x Pixel clock + data output with built in level shifters, multiplexed with 2 of the PWM channels
  • 2x power and ground connections
  • Micro USB port
  • 2x expansion IO headers for future use

Dimensions

  • 1.25 x 2 inches (the USB overhangs by another 0.1 inches or so)

Risks and Challenges

Electronics manufacturing is hard. I would know — I've been doing it for over 10 years. Issues always come up during manufacturing, but there's a lot that can be done to help mitigate them.

  • Production lead times

    Lead times for production quantities of parts can be quite long. 6 to 8 weeks can be typical. However, it is imprudent to order production quantities until your hardware design is finished, otherwise you may be stuck with 1000s of the wrong part. We have an advantage here, in that Chromatron's hardware design is already finished, so we can confidently begin ordering high risk parts (such as processors and connectors) as soon as funds are available.

  • Manufacturing errors

    Manufacturing errors can and do happen. Wrong parts, missing parts, misaligned parts, boards overheated — these are all things that can happen, and are all things I've dealt with in my professional work. Chromatron is being built locally in Austin, TX at a vendor I have past experience with (with excellent results). I can be on site in 20 minutes if issues crop up (there's no substitute for being able to troubleshoot in person). Additionally, I will be doing a lower volume pilot run (100-250 units) to ensure any issues are worked out before committing resources to the full build.

  • Software bugs

    I take software issues very seriously and design my code to be as robust as possible. While much of Chromatron's software is pretty mature at this point, no sane engineer would call their work bug free and I am no exception. The open source nature of the project means you have the opportunity to review and comment on my work, as well as file bug reports (and feature requests!). I use Chromatron 24/7 to run the lighting in my home, so I am personally invested in the quality of the software that runs it.

  • Communication

    Almost everyone on Kickstarter has heard of projects plagued by communications issues. Chromatron is not going to be one of them. I will communicate any and all issues that come up and what I'm doing to handle them, along with general progress of the various parts of the project. It is my hope that Chromatron can become a sustainable business in the future, and I believe the only way that can happen is to earn the trust of my community of backers.
